hexo静态博客部署到github

  • -Hexo
2016-12-10 clara

经过一段时间的测试和试验，今天终于测试通过，调好样式及图片等内容。非常感谢凹凸实验室提供的o2主题，视觉效果很棒，是自己喜欢的类型。当然，也碰到了一些坑，代码拉到本地并不能直接使用，通过不断调试来解决报错问题。

ok，下面看下如何部署到github ？

Quick Start

新建一个github pages

打开Github，新建一个repository。仓库名一定要是 xxx.github.io ，这样子才代表它是一个pages。xxx可以替换成你喜欢的单词。它其实就是一个二级域名，你可以自己购买域名，然后cname过来。就会很有逼格，拥有自己的独立域名。

上传代码到github

关于如何在本地搭建hexo，我就不在这里说了，不然篇幅会很长。

新建好之后，获取到Github的repo地址。

来到hexo项目根目录，打开_config.yml 文件，拖到最底部找到代码deploy:，然后更改成

需要注意的是 repository 后面就是你的repo地址，这里要替换正确，否则上传代码就会出错。

ok，完了后执行 hexo deploy

看到没有报错，就证明执行成功。打开你的Github，你会发现代码已经上传ok。

访问你的博客

激动人心的时刻到了，在浏览器中输入 https://clara.github.io 就会惊喜的发现，博客正常显示啦。棒棒的。

需要注意的是现在github pages 不支持http访问，只能https。

可能遇到的坑

error deployer not found:git

假如你遇到这个问题，说明hexo-deployer-git没有安装，只需要安装下就好了。
