理解前端模块化

  • 模块化
2017-01-18 clara

前言

模块化在2016来说已经不是一个很新鲜的东西呢？大家都在使用模块化开发前端代码，支持模块化的构建工具也被大家广泛使用。

为什么需要模块化

这主要得益于前端技术的发展及页面复杂性的增加。有了模块化，我们可以更方便地使用别人的代码，想要什么功能，就加载什么模块，十分便捷。以前我们的web页面只用来展示静态内容，撑死加上几个飞来飞去的动画，现在前端内容越来越丰富，可以播放视频，可以协同工作，还可以玩游戏。这就导致了前端代码量剧增。当代码行蹭蹭往上涨时，模块化思想自然而然地出来了。

模块化发展

一、CommonJS规范

在commonjs中，有一个全局方法require()，用于同步加载模块，然后通过exports或module.exports来导出所需要暴露的接口。

缺点：

1、同步的模块加载方式不适合在浏览器环境中，同步意味着阻塞加载

2、不能非阻塞的并行加载多个模块

二、AMD规范

它采用异步方式加载模块，模块的加载不影响它后面语句的运行。所有依赖这个模块的语句，都定义在一个回调函数中，等到加载完成后，这个回调函数才会运行。

三、CMD规范

在CMD规范中一个模块就是一个文件，全局函数define()用来定义模块，参数可以为函数或对象或者字符串。
